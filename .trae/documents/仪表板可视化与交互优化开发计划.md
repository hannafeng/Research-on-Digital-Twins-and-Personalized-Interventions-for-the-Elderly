# 仪表板可视化与交互优化开发计划

## 项目现状分析
- 现有 React + Vite 前端项目结构
- 已实现基础登录页面和四个角色的仪表板
- 需要提升界面质量，让技术成果被直观感知

## 开发目标
- 构建 3 个核心页面，遵循 "极简交互" 原则
- 使用 React + Ant Design 实现丰富组件
- 集成图表可视化和实时更新功能
- 确保响应式设计，适配电脑/平板

## 开发计划

### 1. 依赖安装与配置
- 安装 Ant Design (`antd`)
- 安装图表库（`echarts` 或 `@ant-design/charts`）
- 安装 WebSocket 客户端库
- 配置 Vite 项目设置

### 2. 护理人员控制台（核心页面）
- 顶部：风险预警栏（红色弹窗显示高风险用户，点击跳转详情）
- 左侧：用户列表（按风险等级排序：红→黄→绿）
- 中间：数字孪生状态面板
  - 健康状态：心率/血压曲线（折线图）
  - 行为状态：近 24 小时活动热力图
  - 情感状态：近 30 分钟情感标签占比饼图
- 右侧：干预建议区（待确认建议，含理由+置信度，确认执行/驳回按钮）

### 3. 家属简化版页面
- 今日状态摘要（活动量、服药情况、异常告警）
- 近 7 天健康趋势图（心率/步数）
- 紧急联系按钮

### 4. 设备管理后台
- 设备列表（设备 ID、类型、在线状态、最后数据时间）
- 离线设备告警（红色标记，支持一键重启指令）

### 5. 交互与可视化优化
- 集成 ECharts 实现各类图表
- 实现 WebSocket 实时更新（风险预警、状态更新）
- 响应式设计，适配不同屏幕尺寸
- 优化用户体验，确保操作流程直观简单

### 6. 部署与测试
- 构建生产版本
- 部署到 Netlify/Vercel
- 测试功能完整性和响应性能

## 技术栈
- React 18
- Ant Design
- ECharts / Ant Design Charts
- WebSocket
- Vite

## 预期交付物
- 完整的 React 前端代码
- 可访问的前端演示环境
- 响应式设计的三个核心页面
- 实时数据更新功能
